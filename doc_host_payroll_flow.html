1. 担当者と役割
担当者役割
店舗リーダー/幹部
ハーレム / セブンエイト / プリンス / オス / 福岡		毎月の勤怠表・売上台帳を締めて提出
南村様給与計算統括。丹下様の計算結果をレビュー＋特殊計算（役職者・入店3ヶ月以内等）
丹下様300名全員分の計算表作成（Excel自動計算）＋役職手当の手入力。税務資料も担当
田中様役員。全従業員の給与明細を確認し、承認または差し戻し
守屋様振込統括。5社分の振込予約・実行。ダブルチェックは南村様に依頼
2. 毎月のスケジュール
時期内容
月末店舗リーダーが勤怠表を最終チェック（締め）
翌月1日 AM10:00店舗が勤怠表＋売上台帳（Excel）を南村様へ提出
1日〜10-11日丹下様が300名全員分の計算表作成 → 南村様がレビュー＋特殊計算
〜12-13日南村様が田中様へ給与明細データ送付 → 田中様が承認 or 差し戻し（1-2日）
13日守屋様が振込予約（全5社） → 南村様がダブルチェック
15日守屋様が振込実行（ネットバンキング）
※現状は約90名が振込、約150名が手渡し。4月分（5月支給）から全員振込に移行確定
3. 給与計算の流れ
Step担当者作業内容
1店舗リーダー勤怠表を締める（月末）
2店舗勤怠表＋売上台帳（Excel）を南村様へ送付（翌月1日 AM10:00締切）
3南村様データを受領し、丹下様に渡す
4丹下様300名全員分の計算表を作成（Excel自動計算）
5丹下様役職手当を手入力で追加
6丹下様計算結果を南村様に返す
7南村様計算結果をレビュー＋役職者・入店3ヶ月以内等の複雑な計算を実施
8南村様チェック済みの給与明細データを田中様に送付
9田中様全従業員の給与明細を確認
10田中様承認 or 差し戻し（差し戻しの場合、南村様が修正→再送付）
11南村様承認済みの給与データを守屋様に送付
12守屋様振込予約を作成（全5社分）
13南村様守屋様の振込予約内容をダブルチェック
14守屋様15日に振込実行（ネットバンキング）
4. 給与の構成（控除・加算）

控除項目

項目備考
源泉徴収10.21%
寮費寮に住んでいる従業員
光熱費寮内の使用分
名刺代会社立替→後から徴収
税理士費用確定申告を税理士に依頼している従業員のみ
罰金遅刻・無断欠勤等（病欠・冠婚葬祭は対象外）
ヘアメイク代 / 撮影キャンセル料-
前払い・貸付金（バンス）貸付返済として給与から差引
未収入金（つけ払い）客の未払い分→ホストから差引。入金確認後に返済

加算項目

項目備考
達成賞等の賞金店舗から毎月一覧が送られる
経費（役職者のみ）一部幹部に月額経費枠あり

補足

  • レギュラーの報酬: 売上に応じてバック率が変動する仕組み。欠勤日数分がマイナスされる
  • 個人システム（役職者）: 人ごとに異なり、毎月変更される従業員もいる。PDFベースのシステム表を紙に印刷して参照しながら計算（デジタル化されていない）
  • シフト勤務者: 時給制が基本
6. 現在の課題
  • データ提出遅延が頻発（1日AM10:00の締切に間に合わない店舗がある）
  • 月末締め→翌月1日提出→15日振込のスケジュールがタイト
  • 田中様への差し戻しが頻繁に発生
  • 修正依頼が個別LINEでバラバラに来る
  • 店舗から後出しの特別要望が来て、その都度再計算が発生
  • 年に数回の「売上100%支給」の適用タイミングが事前に把握困難
  • ホスト部門の給料計算を台帳から自動化したい（守屋様要望）
  • 振込後に給与間違いが判明した場合、不足分は追加振込、過払い分は従業員から返金してもらう対応が発生（レアケース）
  • 個人システムの参照資料がPDF・紙ベースのため、計算時に手間がかかる
7. 未確認事項
  • レギュラーシステム・シフトシステムのサンプル（南村様から提供予定）
  • 計算に使用しているExcelの抜粋（南村様から提供予定）
  • 福岡店舗のデータ提出状況
  • 個人システムのパターン数
  • 正社員について（再確認が必要）
